Five independent review agents each took one seam: trust/consent, knowledge
integrity, retrieval/recall, web security, source-ingestion security. All
objective gates passed at review time: 121/121 tests, strict typecheck,
production build, web smoke test, `npm audit` clean.

The headline: every claim in HANDOFF.md that was checked held up in code —
consent-before-capture is real (no PCM path before the notice resolves and
grace elapses), authorization is deny-by-default on every dimension, the
approval gate has no bypass, approval is atomic via a write-ahead journal
with tested crash recovery, opt-out genuinely erases buffered PCM, SSRF
defence re-validates every redirect hop against full private-range blocklists,
and citation IDs are validated against a disk-verified retrieved set.

## Open findings (priority order)

### Should fix before serious use

1. **Keyword retrieval bypasses the evidence floor** — `src/store.ts`
   `relevanceFloor` filters vector hits only; the FTS/BM25 branch has no
   threshold. When embeddings are down, `search()` silently degrades to
   keyword-only with zero relevance gating, and the default recall output
   doesn't surface the degradation. Add a keyword-score floor and surface
   degraded-mode in the answer.
2. **Host allowlist fails open on `0.0.0.0`/`::` binds** — `src/web/server.ts`
   `validateWebBinding` requires a token for non-loopback but never requires
   `WEB_ALLOWED_HOSTS`; on an unspecified bind with no allowlist, any Host
   header is trusted (DNS-rebinding surface, chains with browser-cached Basic
   auth). Make it fail closed like the missing-token case.
3. **PDF extraction is unbounded** — `src/sources/pdf.ts` has a byte cap but
   no timeout or page cap; a pathological sub-100MB PDF can burn CPU/memory
   with nothing to kill it (every other source type gets timeout+SIGKILL).
4. **Consent notice can post without follow-up** — `src/index.ts`
   `handleRecordStart` couples `channel.send(notice)` with the ephemeral
   `interaction.editReply`; if the second fails the start aborts but the
   public "recording in 10s" notice stays up with no retraction. Fails safe
   (no capture) but is exactly the trust-surface gap v2 was closing.

### Track

5. **`recall.ts` grounding logic has no end-to-end tests** — only its two pure
   helpers are covered. Build the retrieval evaluation set HANDOFF.md already
   recommends (known answers, expected citations, abstention cases).
6. **Citation validation is presence-checked, not entailment-checked** — a
   valid retrieved ID can be attached to a claim that came from a different
   retrieved source. Residual risk, worth stating in docs.
7. **`x-chronicle-workspace` has no identity binding** — one shared web token
   grants every workspace. Fine single-operator; document it as an explicit
   trust boundary (HANDOFF roadmap item 2 covers the real fix).
8. **Ledger events append outside the write lock** — `src/kb.ts` (5 call
   sites): a crash after the KB mutation commits can silently lose the audit
   event; primary data unaffected. Move the append inside the lock/journal.
9. **Subprocess timeout kills the child, not the process group** —
   `src/runtime.ts`: yt-dlp's internal ffmpeg can be orphaned past
   `YOUTUBE_TIMEOUT_MS`. Use detached spawn + kill(-pid).
10. **`extractCitationIds` over-matches any `[bracketed]` text** —
    `src/recall.ts`: echoed brackets can trigger a spurious
    INSUFFICIENT_EVIDENCE. Anchor the regex to the `workspace/…` ID shape.
11. **Search TOCTOU vs concurrent forced rebuild** — `src/store.ts`: a
    `--force` rebuild in another process can make a concurrent search see
    emptied tables and answer INSUFFICIENT_EVIDENCE. Narrow; self-heals.
12. **Minor**: duplicated `REQUIRE_REVIEW` parsing in `src/ingest.ts` vs
    `config.requireReview`; scattered size-limit env reads; UTF-8 assumed
    regardless of charset in `url.ts`; `--max-filesize` advisory for
    streamed yt-dlp downloads; disk-reserve check sampled every 5s;
    session-manifest serialization is in-process only.
